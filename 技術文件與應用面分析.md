## 一、系統概覽

本系統為一套基於 **RAG（Retrieval-Augmented Generation）** 架構的法規問答服務，專門針對台灣《人工智慧基本法》進行條文檢索與問答生成。使用者透過自然語言提問，系統回傳「結論 / 依據 / 建議 / 補充」格式的法律回覆。

```
            │
            └─ 模型推論: gemma4:e4b (via Ollama, 本機執行)
```

### 3.2 規則引擎模組

系統在呼叫 LLM 前，先跑三層規則引擎，命中即直接回傳（速度快、答案穩定）：

| 規則模組 | 涵蓋問題類型 | 對應條文 |
|----------|-------------|---------|
| `boundary_rules.py` | 詢問執照/許可證/罰款等「界外」問題 | 第 1、5、18 條（範圍外說明） |
| `article_rules.py` | 直接詢問主管機關、條文定義 | 第 2 條（主管機關）、第 3 條（定義） |
| `principle_rules.py` | 詢問 AI 原則、高風險系統判斷 | 第 4 條（原則）、第 5、16、17 條 |

### 3.3 答題流程（3 層 Fallback）

```
Rule-based → Generic Article → LLM 生成
     ↓              ↓               ↓
  即時回傳      <1 秒回傳       最長 3 秒（Timeout）
                               超時 → 回傳提示重試
```

## 四、資料流程

### 4.1 文件匯入（ingest）

```
documents/bank_regulations.txt（人工智慧基本法全文）
    │
    ▼
TextLoader → RecursiveCharacterTextSplitter
    chunk_size=500, chunk_overlap=150
    │
    ▼
HuggingFace Embedding (MiniLM)
    │
    ▼
ChromaDB（chroma_db/chroma.sqlite3）
```

### 4.2 查詢流程

```
使用者問題
    │
    ├─ 有「第 X 條」→ article_lookup 直接抓原文區塊（精確）
    │
    └─ 無條文號 → ChromaDB similarity_search（向量語意搜尋）
    │
    ▼
組裝 context（最多 max_k=1~3 個文件區塊）
    │
    ▼
嘗試 Rule-based → Generic → LLM（依優先順序）
    │
    ▼
answer_normalizer 正規化輸出
    │
    ▼
回傳 QueryResponse { answer, source, duration }
```

### 4.3 快取機制

- 快取範圍：**僅 LLM 生成的答案**（rule-based 不快取，因為速度已夠快）
- 快取 Key：`SHA-256(question + context)`
- TTL：30 分鐘（`CACHE_TTL_SECONDS = 1800`）
- 儲存位置：in-memory dict（重啟後清空）

## 六、應用面分析

### 6.1 適用場景

| 場景 | 說明 |
|------|------|
| 法規諮詢助手 | 讓一般民眾、企業用自然語言查詢 AI 基本法內容，降低法律閱讀門檻 |
| 內部合規工具 | 企業法務或政府機關快速確認特定條文要求，判斷業務是否符合規範 |
| 教學輔助工具 | 法律、資訊相關科系課堂問答練習 |
| RAG 原型參考 | 作為本機 RAG 系統的工程範本，可替換法規文件擴展到其他法律領域 |

### 6.2 系統優點

**架構面：**
- 完全本機執行（Ollama + ChromaDB），無資料外洩風險，適合法律敏感資料
- 三層 Fallback 設計使大多數問題不需呼叫 LLM，回應速度快
- 串流 SSE 端點改善使用者等待體驗
- 模組化設計清晰，各模組職責單一，易於維護與測試

**準確性面：**
- 精確條文號查詢（如「第 2 條」）走直接文字比對，不依賴向量搜尋，準確率高
- 規則引擎針對常見問題類型預設答案，避免 LLM 幻覺
- `answer_normalizer` 確保輸出格式一致，無多餘符號

### 6.3 現有限制與改善建議

| 問題 | 說明 | 建議改善方向 |
|------|------|------------|
| **單一法規** | 目前只有人工智慧基本法一份文件，覆蓋範圍有限 | 擴充 `documents/` 加入子法、施行細則、相關法規 |
| **快取非持久** | 重啟後快取清空，熱點問題需重新生成 | 改用 Redis 或 SQLite 做持久化快取 |
| **Embedding 模型較小** | `MiniLM-L12-v2` 為輕量模型，語意理解深度有限 | 可評估換用 `bge-m3` 或 `text-embedding-3-small` |
| **max_k=1 過小** | `load_query_state` 預設 `max_k=1`，複雜問題可能漏抓相關條文 | 針對複雜問句調高至 3~5 |
| **無評測機制** | 沒有自動化測試集驗證問答正確率 | 建立 Golden Dataset（問題 + 預期條文 + 預期答案）做回歸測試 |
| **LLM Timeout 偏短** | 3 秒 timeout 在模型負載高時容易觸發 | 加入動態 timeout 或佇列機制 |
| **規則引擎覆蓋率低** | 目前只有約 6 種規則，大多數問題仍走 LLM | 擴充規則引擎，對 20 條逐條建立對應問答樣板 |
